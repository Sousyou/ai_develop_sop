# CURRENT Template

Use this template for `.dev_sop/control/CURRENT.md`.
This file should be the recovery-first artifact for humans returning to the project.
Keep it short.
Do not turn it into a project board, work log, or task ledger.
Update it only when the current phase, current target, active slice, source of truth, or next durable actions changed.

## Do not put here
- running progress updates
- per-task implementation notes
- validation transcripts
- inventories of every changed file

## Current phase
What phase is the project currently in?

## Current target
What narrower objective is being advanced now?

## Active slice
What reviewable slice is currently active or was most recently completed?

## In scope now
What is currently included?

## Out of scope now
What is explicitly excluded right now?

## Current source of truth
List only the files that are authoritative right now.
Prefer role files, the rule index, and the active spec, not an exhaustive file inventory.

## Frozen decisions
Which durable project-level choices are currently in force?

## Latest experiment
Which experiment outcome matters most right now, if any?

## Next 3 actions
What are the next few concrete actions?

## Risks to watch
What could make the current view stale or misleading?
