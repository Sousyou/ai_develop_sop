# Skill Template

Use this structure when creating a new skill document.

```md
# <skill_name>

- version:
- status:
- role:

## Trigger

- when should this skill be used

## Inputs

- required inputs

## Procedure

1. step one
2. step two
3. step three

## Outputs

- expected outputs

## Boundaries

- what this skill does not handle
```
