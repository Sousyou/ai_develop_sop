# Boundary Rules

- **Version**: `v2.0`
- **Status**: official rule
- **Role**: keeps execution inside the correct boundary

---

## Default Rules

1. define in-scope and out-of-scope before implementation
2. do not expand from the current task into adjacent work without an explicit decision
3. if a boundary is unclear, stop and clarify through the correct layer
4. product-space changes and SOP-governance changes must not be mixed casually
5. if runtime boundaries drift, correct the entry and mount documents first
