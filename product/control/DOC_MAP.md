# DOC MAP

This file is the starter seed `DOC_MAP.md` for copied projects.

## Start order

1. `control/CURRENT.md` for starter-seed current state.
2. `core/rules/rule-index.md` for baseline rules.
3. `project-rules/rule-index.md` for the copied project's rule layer.
4. `AGENTS.md` for Codex or `CLAUDE.md` for Claude Code.
5. the active task spec in `project-specs/*.md` for execution scope.
6. `VERSION.md` and `upgrades/*` for package version or migration questions.

## Document roles

- `core/*`
  Starter-owned reusable rules, templates, guides, examples, and skills.

- `control/*`
  Starter seed recovery documents for copied projects.

- `project-rules/*`
  Copied-project rule surface.

- `project-specs/*`
  Copied-project execution contracts.

- `project-facts/*`
  Copied-project stable reusable context.

- `upgrades/*`
  Version-targeted package upgrade notes.
