# Project Rule Exceptions

This file records approved exceptions to repository hard rules.

Until an exception is listed here, the default answer is that no exception exists.

## Active exceptions

None.

## Recording format

When an exception is approved, add an entry with:

- `Exception ID`
- `Rule`
- `Scope`
- `Reason`
- `Approved By`
- `Approved On`
- `Expiry Or Review Trigger`
- `Status`

Any task spec that relies on an exception must list the `Exception ID` under `Approved Exceptions`.
