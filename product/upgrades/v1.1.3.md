# Upgrade To 1.1.3

## Intent

Remove the legacy typo compatibility path for the optional sandbox workspace flag.

This version keeps root `sandbox/` as the only canonical isolated workspace and treats the old typo-only CLI spelling as unsupported.

## Breaking Change

After normalization to `1.1.3`:

- `dev-sop init --with-sandbox` and `dev-sop update --with-sandbox` ensure root `sandbox/` exists as a directory
- the CLI no longer accepts the old typo spelling for the sandbox flag
- if the old typo spelling is used, the CLI now fails fast and tells the operator to switch to `--with-sandbox`
- root `sandbox/` remains the SOP-routed isolated workspace for disposable tests, experiments, or temporary validation environments
- the existing `.dev_sop/*` ownership and entrypoint merge rules remain in force

## Applies To

- copied projects already on `1.1.2`
- copied projects or automation wrappers that still invoke the old typo spelling for the sandbox flag
- copied projects that still have legacy disposable material under root `sanbox/` from `1.1.1`

## Upgrade Procedure

1. read the current project's `.dev_sop/VERSION.md`
2. search local scripts, documentation, aliases, and automation wrappers for the old sandbox flag spelling and replace each use with `--with-sandbox`
3. if the project needs the isolated test workspace, run `dev-sop update /path/to/project --yes --with-sandbox`
4. if the project still has a root `sanbox/` directory from `1.1.1`, rename or merge it into root `sandbox/` before relying on the current convention
5. if isolated experimental work was previously spread across ad hoc root `test*`, `tmp*`, or `playground*` directories, consolidate that disposable material into `sandbox/` when safe
6. if a root file named `product`, `dev`, or `sandbox` blocks the required directory path, rename or remove the blocking file before rerunning the update
7. if `AGENTS.md` or `CLAUDE.md` changes during the upgrade, start a fresh Codex or Claude Code thread before relying on the updated instructions

## Validation

- root `product/` exists as a directory
- root `dev/` exists as a directory
- if the isolated test workspace is needed, root `sandbox/` exists as a directory
- the current project's scripts and automation wrappers no longer invoke the removed typo spelling for the sandbox flag
- no conflicting root files block `product/`, `dev/`, or requested `sandbox/`
- any legacy disposable material under root `sanbox/` was migrated, merged, or intentionally retired before cleanup
- no new ad hoc root `test*` directories were introduced for isolated disposable testing
- `.dev_sop/VERSION.md` exists and states `1.1.3`
- copied-project entrypoints still reference `.dev_sop/*` rather than source-repo-only paths
