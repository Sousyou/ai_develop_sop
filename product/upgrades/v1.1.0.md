# Upgrade To 1.1.0

## Intent

Standardize the copied-project root workspace layout.

This version makes `product/` and `dev/` first-class bootstrap directories and keeps `sandbox/` as an explicit optional workspace for disposable tests or experiments.

## Contract

After normalization to `1.1.0`:

- `dev-sop init` and `dev-sop update` ensure root `product/` and `dev/` exist as directories
- `dev-sop init --with-sandbox` and `dev-sop update --with-sandbox` also ensure root `sandbox/` exists as a directory
- root `product/`, `dev/`, and optional `sandbox/` are project-owned working directories outside `.dev_sop/*`
- root `product/` is for final product outputs, deliverables, or release-ready artifacts
- root `dev/` is for in-progress development artifacts, drafts, and intermediate outputs
- root `sandbox/` is for disposable tests, experiments, or temporary validation environments
- the existing `.dev_sop/*` ownership and entrypoint merge rules from `1.0.0` remain in force

## Applies To

- copied projects already on `1.0.0`
- copied projects with the standardized `.dev_sop/*` structure but missing one or both root workspace directories

## Upgrade Procedure

1. read the current project's `.dev_sop/VERSION.md`
2. run `dev-sop update /path/to/project --yes`
3. if the project needs a disposable sandbox, run `dev-sop update /path/to/project --yes --with-sandbox`
4. if a root file named `product`, `dev`, or `sandbox` blocks the required directory path, rename or remove the blocking file before rerunning the update
5. if `AGENTS.md` or `CLAUDE.md` changes during the upgrade, start a fresh Codex or Claude Code thread before relying on the updated instructions

## Validation

- root `product/` exists as a directory
- root `dev/` exists as a directory
- if `--with-sandbox` was used, root `sandbox/` exists as a directory
- no conflicting root files block `product/`, `dev/`, or requested `sandbox/`
- `.dev_sop/VERSION.md` exists and states `1.1.0`
- copied-project entrypoints still reference `.dev_sop/*` rather than source-repo-only paths
