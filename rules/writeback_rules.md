# 回写规则

- **版本**：v1.0
- **定位**：固定稳定事实写回与任务结束检查动作，避免重要结论停留在对话中。

---

## 1. 默认要求

任务完成后，必须检查：

1. 是否需要写回 facts
2. 是否需要更新 SOP
3. 是否需要更新规则层
4. 是否达到 skill 沉淀条件

## 2. 事实回写

- 已确认且会持续影响后续开发的结论，应写入 `docs/facts/*`。
- 新增 facts 文档时，必须登记到 `docs/facts/facts_index.md`。

## 3. SOP / 规则层更新

- 若任务暴露出稳定流程缺口，应更新对应 SOP。
- 若任务暴露出持续执行护栏缺口，应更新 `rules/*`。

## 4. Skill 沉淀

若某类任务重复出现，且输入输出边界、执行方式与验证标准都已稳定，应进入 skill 判断：

- 更新 `skill_policy.md`
- 登记 `skill_registry.md`
- 必要时创建新的 skill 文档

## 5. 规则要求

- 不得只在总结中模糊提及“后续再看”。
- 若当前任务不适合顺手更新文档，必须明确记录应更新的目标文档。
