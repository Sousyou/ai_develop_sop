# Core Rule Index

This file is the authoritative entry for the starter-owned baseline rule layer.

Use it before reading project-generated rules so the shared SOP defaults are clear first.

## Start order

1. Read `workflow.md` for the default execution contract.
2. Read `boundaries.md` for scope and layer discipline.
3. Read `validation.md` for baseline validation expectations.
4. Read `writeback.md` for baseline write-back routing.
5. Then read `../../project-rules/rule-index.md` for the current project's rule layer.

## Precedence

- `approved exceptions > active task spec > project rules > core rules > entry adapter notes`
- Core rules remain in force unless a project rule tightens, specializes, or explicitly waives them through an approved exception.
- Core rules should not be copied into project rules without a project-specific reason.

## Rule Families

- `workflow.md`
  Default execution flow from plan through closeout.

- `boundaries.md`
  Scope control and layer boundary discipline.

- `validation.md`
  Baseline validation expectations shared across copied projects.

- `writeback.md`
  Baseline routing for control, facts, rules, and reusable skill updates.
