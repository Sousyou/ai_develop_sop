# Project Rule Index

This file is the starter seed rule index for copied projects.

Read `../core/rules/rule-index.md` first.
Then use this file to list any copied-project deltas that remain in force during active work.

## Start order

1. Read `../core/rules/rule-index.md` for the baseline rules.
2. Read `exceptions.md` for approved waivers and the exception recording format.
3. Read any additional copied-project rule files listed below.
4. Read the active task spec in `../project-specs/*.md`.

## Active Project Rules

- No additional starter-seed project rules are required by default.
- Copied projects may add local engineering, validation, release, or compliance rules here when needed.

## Exceptions

- `exceptions.md`
  Approved waivers plus the format for requesting and recording future exceptions.
