# Project Specs Surface

This directory is the starter seed project-spec surface for copied projects.

In this source repository, live product specs should go under `../product/specs/*` instead.

## Purpose

Use `.dev_sop/project-specs/*` in copied projects to hold narrow task specs that:
- bridge plans and implementation
- keep execution reviewable
- bind the current slice to the applicable baseline and project rules
- stay small enough to refine during iteration

## Starter Contract

In the starter package, this directory should contain only:

- this `README.md`
- real copied-project specs after adoption
