# Document Writing Conventions Standard

- **Version**: `v2.0`
- **Status**: official standard
- **Role**: defines the default writing style for SOP documents

---

## Default Conventions

1. prefer short direct sentences
2. prefer explicit scope over implied context
3. prefer stable terminology over stylistic variation
4. separate responsibilities, rules, and examples clearly
5. keep documents easy to scan and cheap to re-read

---

## Recommended Structure

When useful, use this order:

1. title and metadata
2. purpose
3. core rules or core definitions
4. templates or examples
5. conclusion or summary

---

## Language Rules

1. use English as the default SOP language
2. prefer ASCII unless the file truly requires otherwise
3. do not add decorative wording that increases token cost without adding decision value
4. keep list items parallel and decision-oriented

---

## Content Rules

1. do not mix standards into process docs
2. do not mix runtime project state into cross-project standards
3. do not use templates as substitutes for layer definitions
4. do not bury critical rules only inside examples
