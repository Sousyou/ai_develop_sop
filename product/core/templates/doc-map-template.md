# DOC MAP Template

Use this template for `.dev_sop/control/DOC_MAP.md`.
Keep it role-based and short.
Do not turn it into a catalog of every file in the repository.
Update it only when the normal reading order, document families, or overlap rule changed.

## Start order
Which few files should a reader check first, and in what order?

## Document roles
Which major document families answer which questions?

## Priority rule
When documents appear to overlap, which layer wins for:
- current-state questions
- task-execution questions
- project-rule questions
- tool-entry questions
- shared SOP questions
