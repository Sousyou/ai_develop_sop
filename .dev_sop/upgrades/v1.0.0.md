# Upgrade To 1.0.0

## Target Version
`1.0.0`

## Purpose

Use this breaking upgrade when a copied project still uses the old asset-first layout rooted in:

- `.dev_sop/control`
- `.dev_sop/doc`
- `.dev_sop/skill`
- optional `.cursor/rules`

This upgrade moves the project to the split layout rooted in:

- `.dev_sop/core`
- `.dev_sop/tools`
- `.dev_sop/rules`
- `.dev_sop/control`
- `.dev_sop/specs`
- `.dev_sop/facts`
- `.dev_sop/skills`
- `.dev_sop/upgrades`

## Applies To

- copied projects on `0.4.1`
- copied projects on older `0.4.x` versions after their intermediate upgrade notes have landed
- copied projects that still treat `Cursor` as a mainline adapter

## End-State Contract

After the upgrade:

- `.dev_sop/core/*` holds shared rules, guides, templates, and starter-owned skills
- `.dev_sop/tools/codex/*` and `.dev_sop/tools/claude-code/*` are the only mainline tool adapters
- `.dev_sop/rules/*` is the only project rule-expansion surface
- `.dev_sop/control/*` holds `CURRENT.md` and `DOC_MAP.md`
- `.dev_sop/specs/*` holds task specs
- `.dev_sop/facts/*` holds project facts
- `.dev_sop/skills/*` holds project-local skills
- the precedence contract is `active task spec > project rules > tool adapter > core defaults`
- `Cursor` is no longer a mainline adapter in current docs
- `.dev_sop/VERSION.md` exists and states `1.0.0`

## Required Moves

Apply these path migrations:

- `.dev_sop/doc/guides/* -> .dev_sop/core/guides/*`
- `.dev_sop/doc/templates/* -> .dev_sop/core/templates/*`
- starter-owned `.dev_sop/skill/* -> .dev_sop/core/skills/*`
- `.dev_sop/doc/specs/* -> .dev_sop/specs/*`
- `.dev_sop/doc/facts/* -> .dev_sop/facts/*`
- `.dev_sop/control/* -> .dev_sop/control/*`
- project-local rule extensions -> `.dev_sop/rules/*`
- `.cursor/rules/* -> remove; merge any still-needed shared content into .dev_sop/core/rules/* or tool notes`

## Required Changes

1. Rewrite `AGENTS.md` as the Codex entry adapter for the split model.
2. Rewrite `CLAUDE.md` as the Claude Code entry adapter for the split model.
3. Rewrite `.dev_sop/README.md` so it documents the split namespace and precedence contract.
4. Create `.dev_sop/core/rules/*` for shared workflow, boundary, validation, and write-back rules.
5. Create `.dev_sop/tools/codex/*` and `.dev_sop/tools/claude-code/*` as thin adapter layers.
6. Create `.dev_sop/rules/*` and make `rule-index.md` the authoritative project-rule entry.
7. Update the task spec template so it includes:
   - `Applicable Project Rules`
   - `Authoritative References`
   - `Approved Exceptions`
   - `Project Validation Gates`
8. Remove `Cursor` as a mainline dependency from current entry docs, rules, guides, templates, and facts.
9. Update `.dev_sop/VERSION.md` to `1.0.0` only after the migration validates.

## Preserve Rules

Do preserve:

- project-owned real task specs
- project-local rules that still make sense under the split model
- project-local facts and project-local skills
- starter-owned `example-*` reference specs that are still part of the package

Do not preserve:

- mainline `Cursor` dependencies in current docs
- `.cursor/rules/*` as an active dependency
- duplicated rule text spread across root adapters, tool notes, and project rules
- a second parallel tree for specs, facts, or skills under another subtree

## Validation

- the shared kernel roots are `.dev_sop/core` and `.dev_sop/tools`
- the project rule surface is `.dev_sop/rules`
- the shared execution asset roots are `.dev_sop/control`, `.dev_sop/specs`, `.dev_sop/facts`, and `.dev_sop/skills`
- `.dev_sop/rules/rule-index.md` exists
- `.dev_sop/core/templates/task-spec-template.md` includes the new project-rule binding fields
- no mainline docs require `.cursor/rules/*` or `Cursor`
- any remaining old-root or `Cursor` references exist only in historical upgrade notes
- `.dev_sop/VERSION.md` exists and states `1.0.0`

## Completion Rule

The upgrade is complete only when:

1. the copied project matches the split end-state contract
2. preserved project-local customizations are still intact
3. `.dev_sop/VERSION.md` is updated to `1.0.0`
