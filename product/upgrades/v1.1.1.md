# Upgrade To 1.1.1

## Intent

Tighten the isolated test-workspace rule for copied projects.

This version standardizes root `sanbox/` as the only SOP-routed isolated directory for disposable tests or experiments, and it explicitly steers ad hoc root `test*` directories away from the main tree.

## Contract

After normalization to `1.1.1`:

- `dev-sop init` and `dev-sop update` still ensure root `product/` and `dev/` exist as directories
- `dev-sop init --with-sanbox` and `dev-sop update --with-sanbox` ensure root `sanbox/` exists as a directory
- the CLI also accepts the legacy alias `--with-sandbox`, but it now creates `sanbox/`
- root `sanbox/` is the SOP-routed isolated workspace for disposable tests, experiments, or temporary validation environments
- when isolated filesystem testing is needed, use `sanbox/` instead of scattering ad hoc root `test*`, `tmp*`, or `playground*` directories
- the existing `.dev_sop/*` ownership and entrypoint merge rules from `1.1.0` remain in force

## Applies To

- copied projects already on `1.1.0`
- copied projects that adopted the root workspace layout but still use `sandbox/` or ad hoc root test directories for isolated experiments

## Upgrade Procedure

1. read the current project's `.dev_sop/VERSION.md`
2. run `dev-sop update /path/to/project --yes`
3. if the project needs the isolated test workspace, run `dev-sop update /path/to/project --yes --with-sanbox`
4. if the project already has a root `sandbox/` directory from `1.1.0`, rename or merge it into root `sanbox/` before relying on the new convention
5. if isolated experimental work was previously spread across ad hoc root `test*`, `tmp*`, or `playground*` directories, consolidate that disposable material into `sanbox/` when safe
6. if a root file named `product`, `dev`, or `sanbox` blocks the required directory path, rename or remove the blocking file before rerunning the update
7. if `AGENTS.md` or `CLAUDE.md` changes during the upgrade, start a fresh Codex or Claude Code thread before relying on the updated instructions

## Validation

- root `product/` exists as a directory
- root `dev/` exists as a directory
- if the isolated test workspace is needed, root `sanbox/` exists as a directory
- no conflicting root files block `product/`, `dev/`, or requested `sanbox/`
- no new ad hoc root `test*` directories were introduced for isolated disposable testing
- `.dev_sop/VERSION.md` exists and states `1.1.1`
- copied-project entrypoints still reference `.dev_sop/*` rather than source-repo-only paths
