# Upgrade To 1.7.0

Use this upgrade when a copied project already adopted `1.6.0` and you want the current package contract that cleanly separates the source repository's own product runtime from the copyable starter surface.

## Why This Version Exists

The source repository now uses a split model:

- `.dev_sop/*` remains the packaged starter surface
- source-repository live SOP runtime may live outside `.dev_sop/*`

This matters because copied projects should adopt the starter surface, not the source repository's own runtime outputs or repository-local backup layout.

## Intended Outcome

After this upgrade:

- `.dev_sop/*` remains the only packaged starter surface
- copied projects do not expect `.dev_sop/_backup/*` as part of the current starter contract
- starter seed documents under `.dev_sop/control/*` and `.dev_sop/project-*` are understood as copied-project seed surfaces, not as the source repository's own current runtime surface

## Required Changes

1. Update entry docs and package docs so they describe `.dev_sop/*` as the packaged starter surface.
2. Remove `.dev_sop/_backup/*` from the current starter contract.
3. Keep current copied-project outputs under `.dev_sop/project-rules/*`, `.dev_sop/project-specs/*`, and `.dev_sop/project-facts/*`.
4. Keep starter-owned reusable assets under `.dev_sop/core/*`.
5. Update `.dev_sop/VERSION.md` only after the contract validates.

## Validation

Verify all of:

- `.dev_sop/VERSION.md` reports `1.7.0`
- live package docs do not describe `.dev_sop/_backup/*` as part of the current starter contract
- `.dev_sop/control/*` and `.dev_sop/project-*` still exist as copied-project seed surfaces
- `.dev_sop/core/*` still contains the starter-owned reusable assets

## Completion Check

The upgrade is complete when the current package contract talks only about the packaged starter surface and no longer treats repository-local backup as part of that surface.
