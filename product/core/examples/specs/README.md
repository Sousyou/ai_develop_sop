# Spec Examples

This directory holds starter-owned example specs.

Use these files as reference artifacts when you want to see a filled-in task spec, not as the project's live execution surface.

## Boundary

- keep starter-owned example specs here
- keep project-generated real specs under `.dev_sop/project-specs/*`
- keep the reusable blank template under `.dev_sop/core/templates/task-spec-template.md`
