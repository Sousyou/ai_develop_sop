# Upgrade To 1.0.0

## Intent

Define the stable baseline for pure command-line SOP updates.

This is the baseline that future package updates should build on.

## Contract

After normalization to `1.0.0`:

- root `README.md` is project-owned and is not created or managed by the SOP package
- root `AGENTS.md` and `CLAUDE.md` remain SOP-managed entrypoints, but updates preserve `## Project Local Notes`
- root `AGENTS.md` and `CLAUDE.md` keep the canonical heading structure shipped by the starter
- `.dev_sop/README.md`, `.dev_sop/core/*`, and `.dev_sop/upgrades/*` are starter-managed
- `.dev_sop/project-rules/README.md`, `.dev_sop/project-specs/README.md`, and `.dev_sop/project-facts/README.md` are starter-managed surface guides
- `.dev_sop/control/*`, `.dev_sop/project-rules/rule-index.md`, and `.dev_sop/project-rules/exceptions.md` are project-owned after adoption and are not rewritten by default
- the source repository may keep its SOP source under `product/*`, but copied projects should use only `.dev_sop/*`
- any later starter-managed change must be published as a new version rather than mutating this baseline in place

## Applies To

- copied projects already on the current installed structure

## Validation

- the copied project's root `README.md` remains project-owned
- no source-repo-only paths such as `product/*` appear in installed starter-owned docs
- `.dev_sop/README.md`, `.dev_sop/core/*`, and `.dev_sop/upgrades/*` match the current package
- `AGENTS.md` and `CLAUDE.md` retain project-local notes while matching the current starter-managed sections
- `.dev_sop/VERSION.md` exists and states `1.0.0`
