# Upgrade To 1.8.0

## Intent

Align the source repository with a cleaner product-source layout while keeping the installed copied-project namespace unchanged.

## Contract

- the source repository now stores the canonical SOP product tree under `product/*`
- copied projects still install and use the SOP namespace as `.dev_sop/*`
- `_backup` is not part of the current product tree or starter contract
- root `AGENTS.md` and `CLAUDE.md` describe `product/*` as the source tree for the packaged starter

## Migration Notes

For copied projects, no path migration is required inside the target project.

- keep the copied project's live SOP namespace under `.dev_sop/*`
- do not create a `product/` directory in copied projects as part of this upgrade
- keep using existing `.dev_sop/control/*`, `.dev_sop/project-rules/*`, `.dev_sop/project-specs/*`, and `.dev_sop/project-facts/*`

For maintainers of the source repository:

1. treat `product/*` as the canonical editable SOP product tree
2. generate packaged assets from `product/*`
3. keep root `facts/*` as source-repository orientation only
4. do not reintroduce `_backup` into the current product layout

## Validation

- copied-project starter installs still materialize the SOP namespace as `.dev_sop/*`
- packaged assets are generated from `product/*`
- no `_backup` directory is required in the current product tree
- root entry docs point to `facts/*` and `product/*`, not to a source-root `.dev_sop/*`
