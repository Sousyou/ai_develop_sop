# Upgrade To 1.5.0

Use this upgrade when a copied project already adopted `1.4.0` but still keeps optional reference material or starter-repo project artifacts on the canonical mainline surface.

## Why This Version Exists

`1.4.0` clarified the baseline-vs-project split.
`1.5.0` adds a second cleanup rule:

- keep only SOP-required files on the canonical mainline
- move optional references and starter-repo project artifacts into `.dev_sop/_backup/*`

This version is about reducing the starter to the minimum operating surface rather than deleting potentially useful material.

## Intended Outcome

After this upgrade:

- the canonical mainline contains only the required SOP files
- `.dev_sop/core/*` keeps only baseline rules plus the required task-spec template
- `.dev_sop/project-rules/*`, `.dev_sop/project-specs/*`, and `.dev_sop/project-facts/*` stay minimal until real project outputs exist
- optional references and removed project artifacts are retained under `.dev_sop/_backup/*`
- `_backup` is explicitly non-canonical

## Required Mainline Files

Keep on the canonical mainline:

- `README.md`
- `AGENTS.md`
- `CLAUDE.md`
- `.dev_sop/README.md`
- `.dev_sop/VERSION.md`
- `.dev_sop/control/CURRENT.md`
- `.dev_sop/control/DOC_MAP.md`
- `.dev_sop/core/rules/*`
- `.dev_sop/core/templates/task-spec-template.md`
- `.dev_sop/project-rules/README.md`
- `.dev_sop/project-rules/rule-index.md`
- `.dev_sop/project-rules/exceptions.md`
- `.dev_sop/project-specs/README.md`
- `.dev_sop/project-facts/README.md`
- `.dev_sop/upgrades/*`

## Move Out Of The Mainline

Move to `.dev_sop/_backup/<date-or-change-id>/`:

- optional examples
- optional guides
- optional reusable-skill files
- optional templates beyond the task-spec template
- starter-repo project facts such as `scope.md`
- starter-repo project-local rule files such as `engineering.md` or `validation.md` when they are not part of the minimum copied-project contract
- any other retained material that should not remain on the mainline starter surface

## Required Changes

1. Create a non-canonical backup directory under `.dev_sop/_backup/`.
2. Move optional reference files and repo-specific project artifacts into that backup directory.
3. Rewrite mainline entry docs so they describe only the required SOP surface.
4. Rewrite control and project-surface docs so they do not assume optional files exist.
5. Keep the precedence contract unchanged:
   `approved exceptions > active task spec > project rules > core rules > entry adapter notes`
6. Update `.dev_sop/VERSION.md` only after the cleanup lands and validates.

## Validation

Verify all of:

- the canonical mainline contains the required file set and no longer depends on optional references
- `.dev_sop/_backup/*` exists and is described as non-canonical
- `.dev_sop/project-rules/*` does not require local engineering or validation files by default
- `.dev_sop/project-specs/*` contains only `README.md` until real specs exist
- `.dev_sop/project-facts/*` contains only `README.md` until real facts exist
- live mainline docs do not instruct readers to use removed optional files
- `.dev_sop/VERSION.md` reports `1.5.0`

## Do Not Carry Forward

Do not preserve as mainline assumptions:

- that starter-owned examples, guides, skills, or extra templates must remain on the canonical surface
- that repo-specific project facts or local rule files should ship on the mainline starter by default
- that `_backup` is part of the active SOP contract

## Completion Check

The upgrade is complete when:

1. the mainline starter contains only the required SOP surface
2. optional material still exists only as backup
3. copied projects can start from the minimal surface without inheriting this repository's own project artifacts
