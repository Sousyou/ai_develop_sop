# 边界规则

- **版本**：v1.0
- **定位**：固定项目、phase 和单任务的边界要求，防止 AI 在不明确边界的情况下扩散执行。

---

## 1. 边界层级

项目中默认存在三层边界：

1. 项目边界
2. 当前 phase 边界
3. 当前任务边界

## 2. 项目边界

- 由项目目标、总体范围和长期不做事项共同定义。
- 项目边界变化时，应同步更新项目规则层与 facts 层。

## 3. Phase 边界

- 每个 phase 必须有明确目标、非目标和交付边界。
- 不得在同一 phase 中同时承载多个不相干的大目标。

## 4. 任务边界

- 每个任务只承载一个核心目标。
- 任务必须显式定义 in-scope 与 out-of-scope。
- 超出当前任务边界的新工作，默认不做，除非重新定义任务。

## 5. 规则要求

- 不得在边界未明确时创建大规模目录或展开大规模实现。
- 不得用“顺手一起做掉”代替边界判断。
- 若执行中发现边界失真，应转入 `maintenance_recovery_sop.md`。
