# Upgrade To 1.2.0

## Target Version
`1.2.0`

## Purpose

Use this upgrade when a copied project already adopted `1.1.0` but still mixes starter-owned example specs with project-generated real specs under `.dev_sop/specs/*`.

This upgrade makes the split explicit:

- `.dev_sop/specs/*` is reserved for project-generated real specs
- `.dev_sop/core/examples/specs/*` holds starter-owned example specs
- `.dev_sop/core/templates/task-spec-template.md` remains the blank reusable template

## Applies To

- copied projects on `1.1.0`
- copied projects that still keep starter-owned `example-*` specs under `.dev_sop/specs/*`

## End-State Contract

After the upgrade:

- `.dev_sop/specs/*` holds only project-generated real specs plus `README.md`
- starter-owned example specs live under `.dev_sop/core/examples/specs/*`
- `.dev_sop/core/templates/task-spec-template.md` remains the authoritative blank task-spec template
- guides, facts, and upgrade prompts point to the new example-spec location
- `.dev_sop/VERSION.md` exists and states `1.2.0`

## Required Moves

Apply these path migrations:

- starter-owned `.dev_sop/specs/example-* -> .dev_sop/core/examples/specs/*`
- keep project-generated real specs in `.dev_sop/specs/*`
- do not move real project specs into `.dev_sop/core/examples/specs/*`

## Required Changes

1. Rewrite `.dev_sop/specs/README.md` so it defines `.dev_sop/specs/*` as the project-generated spec surface.
2. Create `.dev_sop/core/examples/specs/README.md` as the starter-owned example-spec map.
3. Update namespace, control, facts, and engineering docs so they distinguish templates, examples, and real specs.
4. Update any guide, fact, or prompt that points to starter-owned example specs so it uses `.dev_sop/core/examples/specs/*`.
5. Update `.dev_sop/VERSION.md` to `1.2.0` only after the migration validates.

## Preserve Rules

Do preserve:

- all real project-generated specs under `.dev_sop/specs/*`
- starter-owned example specs as package reference material
- local rule customizations that still make sense under the current model

Do not preserve:

- starter-owned example specs mixed into the live project spec surface
- assumptions that `.dev_sop/specs/*` is both an example library and a live execution directory

## Validation

- `.dev_sop/specs/*` contains only `README.md` plus project-generated real specs
- starter-owned example specs live under `.dev_sop/core/examples/specs/*`
- `.dev_sop/core/templates/task-spec-template.md` still exists
- live mainline references to starter-owned example specs point to `.dev_sop/core/examples/specs/*`
- `.dev_sop/VERSION.md` exists and states `1.2.0`

## Completion Rule

The upgrade is complete only when:

1. project-generated specs remain intact under `.dev_sop/specs/*`
2. starter-owned example specs have moved to `.dev_sop/core/examples/specs/*`
3. `.dev_sop/VERSION.md` is updated to `1.2.0`
