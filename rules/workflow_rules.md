# 执行顺序规则

- **版本**：v1.0
- **定位**：固定 AI 在项目中的标准执行顺序，避免跳步、倒序或边做边漂移。

---

## 1. 适用范围

本规则适用于项目初始化后的所有开发、文档更新、维护和纠偏任务。

## 2. 标准执行顺序

AI 执行任务时，应默认遵循以下顺序：

1. 明确当前目标
2. 明确 in-scope / out-of-scope
3. 明确验证方式
4. 在边界内完成最小实现
5. 执行自检与验证
6. 输出结果总结
7. 判断是否需要 facts 回写
8. 判断是否需要更新规则层 / SOP / skill 层

## 3. 规则要求

- 不得在目标、范围和验证方式未清晰前直接进入实现。
- 不得跳过总结与后续判断环节。
- 若执行中发现当前顺序已不足以支撑任务，应显式触发 SOP 更新判断，而不是私自改变执行顺序。

## 4. 与其他文档的关系

- 任务级执行细节以 `task_execution_sop.md` 为准。
- 若当前任务存在明显偏移，转入 `maintenance_recovery_sop.md`。
- 本规则只固定顺序，不承载项目事实。
